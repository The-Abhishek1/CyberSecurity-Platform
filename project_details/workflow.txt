When You Call /api/v1/hybrid/execute:
python
# 1. Request hits hybrid.py
@router.post("/execute")
async def execute_goal(...):
    # 2. Creates a process_id
    process_id = f"proc_{uuid.uuid4().hex[:12]}"
    
    # 3. Queues it for background processing
    background_tasks.add_task(process_hybrid_execution, process_id, ...)
    
    # 4. Returns immediately with process_id
    return {"process_id": process_id, "status": "pending"}
Background Processing Flow:
text
process_hybrid_execution() [in hybrid.py]
    ↓
scheduler.schedule_execution() [in hybrid_scheduler.py]
    ↓
planner_agent.create_plan() [in planner_agent.py]
    ↓
verifier_agent.validate_dag() [in verifier_agent.py]
    ↓
orchestrator.execute_dag() [in agent_orchestrator.py]
    ↓
tool_router.route_and_execute() [in tool_router.py]
    ↓
worker_pool.execute() [in worker_pool.py]
    ↓
Result stored in memory_service [in memory_service.py]
3. What Each Major File Does
Core Files (src/core/)
config.py - All configuration settings (database, API keys, etc.)

exceptions.py - Custom error types

security.py - Password hashing, JWT tokens, encryption

Scheduler (src/scheduler/)
hybrid_scheduler.py - Main scheduler that manages executions

lifecycle_manager.py - Tracks execution states (pending → running → completed)

budget_tracker.py - Monitors costs

quota_manager.py - Enforces usage limits per tenant

Agents (src/agents/)
planner_agent.py - Breaks down goals into tasks

verifier_agent.py - Checks if the plan makes sense

Memory (src/memory/)
memory_service.py - Stores and retrieves past executions

vector_store.py - For semantic search (find similar tasks)

graph_store.py - For relationships between entities

Orchestrator (src/orchestrator/)
agent_orchestrator.py - Runs tasks in parallel

state_manager.py - Keeps track of running tasks

Tools (src/tools/)
tool_router.py - Decides which tool to use (nmap, nuclei, etc.)

tool_registry.py - List of available tools

Workers (src/workers/)
worker_pool.py - Manages Docker containers

container_manager.py - Creates/stops containers

4. Data Flow Example
text
User says: "Scan example.com"

1. hybrid.py receives request
2. Creates process_id = "proc_abc123"
3. Scheduler starts planning
4. Planner creates tasks:
   - Task 1: Port scan with nmap
   - Task 2: Vulnerability scan with nuclei
5. Verifier checks if possible
6. Orchestrator runs tasks in parallel
7. Tool router picks nmap for task 1
8. Worker pool runs nmap in container
9. Results saved to memory
10. Status updates available via /status/proc_abc123
5. Key Points to Remember
Everything is modular - Each directory has a single responsibility

Files follow the flow - Request → API → Scheduler → Agents → Tools → Workers

State is tracked - Every execution has a status you can check

Everything is logged - Check logs/ directory
